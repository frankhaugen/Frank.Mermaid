# Contributing

We welcome contributions to this project.

You can contribute by:

1. Reporting an issue
2. Fixing an issue
3. Adding a feature
4. Improving the documentation
5. Improving the code quality
6. Adding tests
7. Adding examples
8. Adding a new diagram type

## Limitations

Fundamental changes to the nature of the project, are not accepted. This project is a Mermaid diagram generator, and it will 
not be changed to a general-purpose diagram generator as an example.

Breaking changes to the API are accepted, but they should be well documented in the release notes, and there should be a 
deprecation period before the old API is removed.

Adding a new nuget/assembly like `Frank.Mermaid.Fluent` is encouraged, but it should be a discussion before it is added, 
because it might be better to add it to the existing assembly or maybe it should be a replacement for the existing patterns.

## How to contribute

1. Fork the repository
2. Create a branch
3. Make your changes
4. Create an issue
5. Create a pull request and link to the issue
   - Any contribution credit you want to have should be added to the pull request by you, if not you forfeit any credit
6. Wait for the review

## Code of Conduct

Be nice to each other!
